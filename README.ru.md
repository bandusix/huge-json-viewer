## Зачем это нужно

Большинство текстовых редакторов и онлайн-просмотрщиков JSON **падают или зависают, когда вы пытаетесь открыть большой JSON-файл**, потому что разбирают его целиком в память — файл на 2–3 ГБ раздувается до 15–30 ГБ ОЗУ. **BigJSON** так не делает никогда. Он отображает файл в память (memory-map), за один потоковый проход строит компактный индекс и рисует только те строки, что видны на экране. Поэтому он **открывает многогигабайтный JSON за секунды** и мгновенно ищет по всему файлу, оставаясь в пределах ~1,5–2× размера файла по ОЗУ.

Если вы искали *«как открыть большой JSON файл»*, *«JSON файл слишком большой, чтобы открыть»* или **бесплатную альтернативу Dadroit для Mac или Windows** — это сделано именно для таких задач.

## Возможности

- ⚡ **Открывает JSON на 2–3 ГБ примерно за 3 секунды** — отображение в память, потоковый индекс, виртуализированное дерево
- 🔍 **Поиск по ключам и значениям** — с учётом регистра или без, обычный текст или **regex**, живой счётчик совпадений, переход к следующему/предыдущему с раскрытием пути
- 📤 **Конвертация JSON → CSV или XML** — потоковый экспорт, работающий на многогигабайтных файлах (огромный массив объектов превращается в таблицу)
- 📋 **Копирование и извлечение** — щёлкните правой кнопкой по любому узлу, чтобы **скопировать его ключ, значение, путь** (в стиле jq, например `.users[3].name`) или **скопировать / экспортировать поддерево как JSON**
- 🔗 **Объединение нескольких файлов** — откройте сразу несколько JSON-файлов как одно объединённое дерево с поиском
- 🌳 **Сворачиваемое дерево с подсветкой синтаксиса** — номера строк, направляющие отступов, цвета типов и счётчики дочерних элементов (в стиле Dadroit / jsonviewer.app)
- 📄 Открывает `.json`, `.ndjson` / `.jsonl` (определяются автоматически), `.geojson`, `.txt` — или **вставьте JSON** прямо из буфера обмена
- 🖱️ Перетаскивание файлов, **⌘O / Ctrl+O** для открытия, **⌘V / Ctrl+V** для вставки, **⌘F / Ctrl+F** для поиска, полная навигация с клавиатуры
- 🔔 **Уведомление об обновлении** — ненавязчивая ссылка в строке состояния, когда выходит новая версия (проверяется не чаще раза в день; полностью безопасно в офлайне)
- 🌍 **Интерфейс на 20 языках**, с поддержкой письма справа налево (арабский, урду, панджаби)
- 🖥️ **macOS и Windows** — универсальная сборка для Mac (Apple Silicon M1–M4 + Intel) и Windows 10/11 (x64)
- 🔒 **Ваши файлы никогда не покидают ваше устройство** — ни загрузки файлов, ни сервера, никогда · только анонимная статистика использования, которую можно отключить · крошечное приложение
- 🆓 **Бесплатно и с открытым исходным кодом** (MIT)

## Установка

### macOS

1. **[Скачайте последний `.dmg`](https://github.com/bandusix/huge-json-viewer/releases/latest)** и откройте его.
2. Перетащите **BigJSON** в папку «Программы».
3. Первый запуск: приложение не подписано, поэтому **щёлкните по нему правой кнопкой → «Открыть»** и подтвердите (нужно сделать только один раз).

Требования: macOS 11 (Big Sur) или новее, Apple Silicon или Intel.

### Windows

1. **[Скачайте последний `.exe`](https://github.com/bandusix/huge-json-viewer/releases/latest)** (установщик NSIS `BigJSON_x.y.z_x64-setup.exe`) и запустите его — установка выполняется для текущего пользователя, права администратора не нужны.
2. Сборка не подписана, поэтому если появится **SmartScreen**, нажмите **Подробнее (More info) → Выполнить в любом случае (Run anyway)** (нужно сделать только один раз).

Требования: Windows 10 или 11 (64-бит). WebView2 предустановлен в Windows 11 и актуальных версиях Windows 10; если его нет, установщик подгрузит его автоматически.

Обе сборки работают с JSON-файлами до 4 ГБ.

## Как это работает

JSON-файл на 2–3 ГБ невозможно разобрать в объекты в памяти. Вместо этого ядро на Rust:

1. **Отображает файл в память** (`memmap2`) — страницы подгружаются ОС по требованию, а не хранятся в куче.
2. **Выполняет один потоковый проход токенизатора**, чтобы построить компактный плоский индекс (~23 байта на узел JSON) из смещений в байтах и структуры — без разбора в объекты.
3. **Рисует только видимые строки.** Дерево полностью виртуализировано; раскрытие/сворачивание изменяют список видимых строк, а не материализуют весь документ. **Масштабированная полоса прокрутки** позволяет прокручивать миллионы строк, обходя ограничение браузера на высоту элемента.
4. **Ищет по сырым байтам** — SIMD-поиск подстроки / regex поверх mmap с сопоставлением каждого совпадения с его узлом.

Собрано на **Tauri v2** (бэкенд на Rust + веб-фронтенд), упаковано в `.dmg` размером ~2 МБ (macOS) или установщик NSIS `.exe` (Windows).

## Сборка из исходников

Требования: [Node.js](https://nodejs.org) 20+, [набор инструментов Rust](https://rustup.rs) и [системные требования Tauri v2](https://v2.tauri.app/start/prerequisites/) для вашей ОС (Xcode Command Line Tools на macOS; Microsoft C++ Build Tools + WebView2 на Windows).

```bash
npm install
npm run tauri dev                                       # hot-reloading dev app
npm run tauri build -- --target universal-apple-darwin --bundles dmg   # macOS universal DMG
npm run tauri build -- --bundles nsis                   # Windows installer (run on Windows)
cd src-tauri && cargo test                              # engine tests (serde_json oracle, escapes, NDJSON, search)
```

## Ограничения (v1)

- **Размер файла:** до 4 ГБ (компактные смещения `u32`). Файлы больше отклоняются с понятным сообщением.
- **ОЗУ:** индекс занимает ~23 байта на узел, поэтому файлу на 2–3 ГБ нужно примерно **1,5–2× размера файла** по ОЗУ (сам отображённый в память файл — вытесняемый кэш страниц ОС). Машина с 16 ГБ комфортно справляется с файлами на 2–3 ГБ.
- **Поиск** сопоставляет сырые байты файла (экранированный символ совпадает в своей экранированной форме); поиск без учёта регистра работает только для ASCII.
