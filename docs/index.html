Under the hood

Why it doesn’t choke on gigabytes

1

Memory‑map, don’t load

The file is memory‑mapped and paged in on demand by the OS — never parsed into RAM‑hungry objects.

2

Stream a compact index

One pass builds a flat structural index of byte offsets, so structure is known without holding the data.

3

Render only what you see

The tree is fully virtualized — only the rows on screen are materialized, so scrolling millions of rows stays smooth.

FAQ

Opening large JSON files, answered

How do I open a large JSON file on Mac?

Download Huge JSON Viewer (free), then drag your .json file onto the window or press ⌘O. It memory‑maps the file and builds a streaming index, so a 2–3 GB JSON opens in a few seconds — no crashing, no splitting.

Is Huge JSON Viewer really free?

Yes — 100% free and open source under the MIT license, including commercial use. No paid tiers, no sign‑up, no trial.

How large a JSON file can it open?

Up to 4 GB per file (and 4 GB combined for a multi‑file union). The index uses roughly 1.5–2× the file size in RAM; a 16 GB Mac handles 2–3 GB files comfortably.

Is it a good free alternative to Dadroit?

Yes. It covers the same essentials — a collapsible tree viewer, key and value RegEx search, JSON → CSV / XML export, and unioning multiple files — for free and open source, on Apple Silicon and Intel.

Can it convert JSON to CSV or XML?

Yes. It streams JSON to CSV (an array of objects becomes a spreadsheet) or to XML, and works on multi‑gigabyte files with bounded memory.

Is my data uploaded anywhere?

No. It is 100% offline — no server, no upload, no telemetry. Your JSON never leaves your Mac.
